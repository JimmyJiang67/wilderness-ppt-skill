# 文档解析策略

将用户文档转换为分页方案的规则。

## 解析流程

```
文档输入 → 结构识别 → 页面类型匹配 → 分页方案 → 用户确认
```

## 结构识别规则

### 标题层级映射

| Markdown | 页面类型 | 说明 |
|----------|----------|------|
| `# 标题` | Cover | 文档主标题 → 封面页 |
| `## 标题` | 新页面 | 二级标题 → 新slide开始 |
| `### 标题` | 页面内卡片 | 三级标题 → 当前页内的卡片标题 |
| `- 要点` / `1. 列表` | 内容项 | 列表项 → 页面要点 |

### 内容特征识别

| 特征 | 匹配页面类型 | 判断规则 |
|------|-------------|----------|
| 数字/百分比/指标 | Data | 包含 `[0-9]+[%w+]?` 格式 |
| 对比词（vs/对比/区别） | Case | 包含 "vs"、"对比"、"区别"、"差异化" |
| 图片路径/描述 | Gallery | 包含图片文件名、`.jpg/.png` 等 |
| 章节列表 | Contents | 多个同级标题连续出现 |
| 流程/步骤说明 | Content-Split | 包含 "第一步"、"流程"、"步骤" 等 |

## 分页方案生成

### 内容密度限制

| 页面类型 | 最大内容数 | 超限处理 |
|----------|-----------|----------|
| Cover | 1主标题+1副标题 | N/A |
| Contents | 8个章节项 | 超出分多页目录 |
| Content-Split | 6个要点 | 超出则分多页 |
| Data | 4个数据指标 | 超出则分多页 |
| Case | 3个对比维度 | 超出则分多页 |
| Gallery | 6张图片 | 超出则分多页 |

### 分页规则

1. **二级标题触发新页**：每个 `##` 默认创建新slide
2. **列表项自动分页**：同层级列表超过6项时，每6项分一页
3. **段落长度检测**：单段落超过150字时，考虑拆分为多页或精简
4. **混合内容处理**：文字+图片 → Gallery；文字+数据 → Data；文字+对比 → Case

## 用户确认流程

1. **生成分页方案**后，向用户展示：
   - 总页数
   - 每页类型和标题
   - 内容概要（前3个要点）

2. **询问确认**：
   - "分页方案如上，是否需要调整？"
   - 选项：确认生成 / 调整页面 / 增删内容

3. **确认后**进入Phase 2生成HTML

## 特殊处理

### 图片处理
- 识别文档中的图片路径（`.jpg`, `.png`, `.webp`, `.gif`）
- 图片数量 > 0 → 创建Gallery页或作为Content-Split的配图
- 无图片路径但内容描述视觉 → 使用占位符，提示用户后续插入

### 表格处理
- 简单2列对比 → Case页面
- 多行数据 → Data页面或表格形式展示
- 复杂表格 → 转换为要点列表

### 代码/技术内容
- 代码片段 → Content-Split页面，使用等宽字体展示
- 技术参数 → Data页面展示

## 示例解析

### 输入文档
```markdown
# AIGC培训课程

## 目录
1. AI基础
2. 人物生成
3. 场景生成

## 人物生成

### 核心要点
- 身份定义
- 风格选择
- 细节修饰

生成60+视频，播放量300W+，满意度4.8/5
```

### 输出分页方案
```json
[
  { "type": "cover", "title": "AIGC培训课程" },
  { "type": "contents", "items": ["AI基础", "人物生成", "场景生成"] },
  { "type": "content-split", "title": "人物生成", "points": ["身份定义", "风格选择", "细节修饰"] },
  { "type": "data", "title": "成果数据", "metrics": [{"value":"60+","label":"视频数"},{"value":"300W+","label":"播放量"},{"value":"4.8","label":"满意度"}] }
]
```
